Проект построен на PHP 8.4. Контейнеризация и оркестрация выполнены на базе Docker.

## 🛠 Стек технологий
* **Язык программирования:** PHP 8.4 (режим строгой типизации `strict_types`)
* **База данных:** PostgreSQL 16 
* **Веб-сервер:** Caddy Server v2 
* **Инструменты развертывание:** Docker Compose + Makefile

## 🚀 Быстрый старт (Установка и запуск)

Перед развертыванием необходимо создать локальный файл конфигурации окружения. На основе демонстрационного шаблона (docker/.env.example) создайте рабочий файл параметров:
```bash
cp docker/.env.example docker/.env
```
*При необходимости отредактируйте `docker/.env`, чтобы изменить стандартные порты, имя базы данных или пароли.*

Дальнейший запуск возможен двумя альтернативными способами: автоматическим (через `Makefile`) или ручным (напрямую через `docker compose`).

---

### Вариант А. Автоматический запуск (Рекомендуемый)

Все команды выполняются из **корня** папки `myproject` через утилиту управления `make`:

```bash
make init
```

---

### Вариант Б. Ручной запуск через Docker Compose

Если на вашей системе не установлена утилита `make`, вы можете поднять окружение стандартными командами Docker. Для этого необходимо зайти в конфигурационную подпапку:

1. **Перейдите в папку с Docker-конфигурацией:**
   ```bash
   cd docker
   ```
2. **Запустите сборку и развертывание контейнеров:**
   ```bash
   docker compose up -d --build
   ```

---

### 2. Доступ к интерфейсам
После успешного запуска перейдите в браузер:
* **Веб-приложение (Импорт и Аналитика):** [https://localhost](https://localhost)
* **Управление базой данных (Adminer):** [http://localhost:7011](http://localhost:7011)
  * *Параметры входа в Adminer:* Тип БД: `PostgreSQL`, Сервер: `database`, Пользователь/Пароль/БД берутся из файла `docker/.env`.

### 📝 Описание переменных окружения (`.env.example`)

Конфигурационный файл разбит на три логических блока:

#### 1. Системные настройки Docker (`Docker Container`)
* `CONTAINER_NAME=sdt-ctn` — базовый префикс для именования создаваемых Docker-контейнеров в системе.
* `COMPOSE_PROJECT_NAME=sdt-prj` — уникальное имя Docker Compose проекта (используется для изоляции сети).
* `SITE_DOMAIN=localhost` — доменное имя, по которому Caddy-сервер будет отдавать сайт и для которого автоматически сгенерирует локальный SSL-сертификат.

#### 2. Конфигурация СУБД (`База данных (PostgreSQL)`)
* `POSTGRES_VERSION=16` — версия официального Docker-образа PostgreSQL, которая будет скачана и запущена.
* `POSTGRES_DB=sdt_db` — имя создаваемой базы данных при первичной инициализации СУБД.
* `POSTGRES_USER=sdt_user` — имя суперпользователя для подключения к базе данных.
* `POSTGRES_PASSWORD=sdt_user.pass` — пароль для созданного пользователя.
* `POSTGRES_HOST=database` — сетевой хост для подключения (внутри Docker-сети это имя сервиса `database`).
* `DB_PGSQL_PATH=./docker-data/dockDB/pgsql-data/` — путь на вашем физическом компьютере, куда PostgreSQL будет сохранять бинарные файлы таблиц для обеспечения постоянного хранения данных (persistent storage).

#### 3. Настройки инструментов управления
* `ADMINER_PORT=7011` — внешний порт хост-машины, по которому в браузере будет доступна веб-панель Adminer для управления базой данных (`http://localhost:7011`).

## 📋 Инструкция по использованию

### 1. Модуль импорта файлов (Вкладка "Upload")
Преддназначен для загрузки текстовых файлов с данными о заказах. Разделитель полей — точка с запятой (`;`).
* **Структура строки в файле:** `ID товара;ID клиента;Комментарий;Статус заказа;Дата заказа`
* **Особенности работы:**
  * Если при отправке формы файл не выбран, система автоматически использует демонстрационный файл по умолчанию (`app/orders.txt`).
  * Скрипт валидирует типы данных, структуру колонок и допустимые статусы (`new`, `complete`).
  * Все невалидные строки изолируются и записываются в файл `app/invalid_orders.txt`, после чего выводятся списком на экран.

### 2. Модуль аналитики (Вкладка "Query")
Позволяет формировать четыре отчета.
1. **Запрос 1:** Поиск клиентов, не совершавших заказы за последние 7 дней.
2. **Запрос 2:** Топ-5 клиентов, сделавших наибольшее количество заказов.
3. **Запрос 3:** Топ-10 клиентов, совершивших покупки на самую большую общую сумму (на основе стоимости товаров `merchandise.price`).
4. **Запрос 4:** Поиск товаров, по которым не было ни одного доставленного заказа со статусом `complete`.

### 3. Задача (Вкладка "Task")
Описание "тестового задания".

## 🛠 Дополнительные команды управления (`Makefile`)

* `make up` — запустить контейнеры в фоновом режиме.
* `make down` — остановить проект и удалить контейнеры.
* `make build` — принудительно пересобрать Docker-образы без использования кэша.
* `make shell` — войти в терминал контейнера PHP для ручного тестирования (например, выполнения `php php/import.php`).
* `make db-clean-import` — полностью стереть файлы СУБД на хосте, сбросить автоинкременты и переинициализировать структуру базы данных с нуля.
* `make clear` — полная очистка Docker-окружения и удаление всех томов (volumes).
